# LinkNest 核心流程测试

## 测试范围

第 14 步覆盖 MVP 主链路：

- 游客浏览附近图书
- 登录 / 注册
- 发布图书
- 申请借阅
- 出借者处理申请
- 借阅和归还状态流转
- 添加好友
- 消息通知
- 隐私和举报入口

## 环境

本地 Web 预览：

```bash
npm run dev
```

如果端口被占用，Vite 会自动切到下一个端口。以终端显示的 `Local` URL 为准。

当前也可以用：

```text
http://127.0.0.1:5176/
```

## 自动检查

每次提交前至少运行：

```bash
npm run typecheck
npm run build
```

如果本地 dev server 已启动，再检查：

```bash
curl -I http://127.0.0.1:5176/
```

预期返回：

```text
HTTP/1.1 200 OK
```

## 手动测试清单

### 1. 游客浏览

步骤：

1. 打开发现页。
2. 切换地图 / 列表。
3. 搜索书名或作者。
4. 切换分类：全部、文学、小说、童书。
5. 切换距离：1km、3km、5km、10km。
6. 点击一本书进入详情。

预期：

- 游客可以浏览附近图书。
- 距离显示为模糊距离。
- 无结果时显示空状态。
- 书籍详情不展示精确地址。

### 2. 登录 / 注册

步骤：

1. 点击“我的”。
2. 点击登录。
3. 使用默认演示邮箱和密码登录。
4. 退出登录。
5. 再进入注册页。

预期：

- 未登录时显示登录引导。
- 未配置 Supabase 时进入本地演示登录。
- 登录后“我的”显示用户资料。
- 退出登录后回到游客状态。

### 3. 发布图书

步骤：

1. 登录。
2. 进入书架。
3. 点击“添加新书”。
4. 填写书名、作者、分类、语言、成色、年份、简介。
5. 选择“上架共享”。
6. 点击发布。

预期：

- 发布成功后回到书架。
- 新书出现在“我的共享”。
- 新书可在发现页看到。
- 未配置 Supabase 时保存为本地演示图书。
- 配置 Supabase 且真实登录时，新书写入 `books` 表，刷新后仍保留。

### 4. 申请借阅

步骤：

1. 进入发现页。
2. 打开一本可借图书。
3. 点击“申请借阅”。
4. 填写留言。
5. 发送申请。

预期：

- 未登录时先进入登录。
- 登录后可以发送申请。
- 发送成功后显示申请已发送。
- 书籍状态变为处理中。
- 消息页出现借阅通知。
- 配置 Supabase 且真实登录时，`borrow_requests` 产生记录，`books.status` 由 trigger 同步为 `pending`。

### 5. 出借者处理申请

步骤：

1. 登录演示账号。
2. 进入书架。
3. 切到“我出借的”。
4. 找到待处理申请。
5. 点击同意或拒绝。

预期：

- 同意后状态变为待交接。
- 拒绝后状态变为已拒绝。
- 消息页新增状态通知。
- 书籍状态随借阅状态变化。

### 6. 借阅和归还

步骤：

1. 进入书架。
2. 切到“我借入的”。
3. 对已同意申请点击“确认已拿到”。
4. 再点击“申请归还”。
5. 切到“我出借的”。
6. 点击“确认已归还”。

预期：

- 状态按顺序流转：待交接 -> 借阅中 -> 待确认归还 -> 已归还。
- 已归还后图书恢复可借。
- 每次状态变化都有消息通知。
- 配置 Supabase 且真实登录时，刷新页面后仍能在书架看到当前借阅状态。

### 7. 好友系统

步骤：

1. 进入邻居页。
2. 点击一个非好友的“+ 好友”。
3. 查看好友申请发送成功页。
4. 回到邻居页。
5. 处理待处理好友申请：同意 / 拒绝。
6. 进入出借者详情页查看按钮状态。

预期：

- 好友状态显示为等待通过、好友、待处理等。
- 同意后成为好友。
- 拒绝后状态变化。
- 消息页出现好友通知。
- 配置 Supabase 且真实登录时，`friendships` 表保存状态，刷新页面后仍保留。

### 8. 消息通知

步骤：

1. 触发借阅申请或好友申请。
2. 进入消息页。
3. 点击借阅通知。

预期：

- 底部消息 Tab 出现未读点。
- 消息页显示通知列表。
- 进入消息页后未读点清除。
- 点击图书相关通知可进入图书详情。

### 9. 隐私和举报

步骤：

1. 打开书籍详情。
2. 查看隐私提示。
3. 未登录时点击举报入口。
4. 打开出借者详情。
5. 登录后点击举报入口。
6. 进入我的页面，查看后端联调状态。
7. 调整资料可见性和可见范围。

预期：

- 不展示精确地址。
- 未登录时会进入登录页。
- 登录后举报入口可提交，并显示已收到状态。
- 配置 Supabase 且真实登录时，`reports` 表产生 open 状态记录。
- 配置 Supabase 且真实登录时，收藏会写入 / 删除 `favorites` 表。
- 未配置 Supabase 或演示登录时，举报保存为本地演示记录。
- 我的页显示 Supabase 配置、会话、图书数据源和举报写入目标。
- 隐私设置 UI 可切换。

- 尚未接入真实端到端自动化测试。

- Supabase RLS 需要在真实项目中用匿名用户、普通用户和 service role 分别测试。
