# Link Nest App 重新开发规划

## 1. 产品定位

Link Nest 是一个面向邻里社区的图书共享 App。核心体验是：

- 用户无需登录即可发现附近可借书籍。
- 只有在申请借阅、创建书架、添加好友、沟通等关键动作时才要求登录。
- 以「附近书籍」为入口，围绕书籍、出借者、借阅申请、好友、消息和我的书架形成闭环。

建议第一版目标是做成一个稳定、完整、可上线的 MVP，而不是一次性做满所有高级能力。

## 2. MVP 范围

### 必须完成

- 附近发现页：地图模式、列表模式、距离筛选、分类筛选。
- 书籍详情页：封面、标题、作者、距离、可借状态、出借者、评分、简介、用户评论。
- 借阅申请流程：选择书籍、填写留言、发送申请、状态跟踪。
- 登录与注册：手机号、邮箱或第三方登录至少支持一种。
- 我的书架：添加图书、编辑图书、发布到共享书架、查看我出借和我借入的书。
- 消息中心：借阅通知、好友消息、系统消息。
- 好友流程：申请好友、同意好友、好友列表。
- 我的页面：个人信息、地址/范围设置、隐私设置、借阅历史、关于 App。

### 第一版可以延后

- OCR / ISBN 自动识别。
- 高级推荐算法。
- 多社区切换。
- 押金、支付、信用体系。
- 复杂评价体系和举报审核后台。
- 离线缓存和多端同步优化。

## 3. 推荐技术路线

如果目标是快速做出 iOS/Android 双端：

- 前端：React Native + Expo
- 路由：Expo Router
- UI：React Native Paper 或 Tamagui，也可以自定义轻量组件库
- 地图：react-native-maps
- 状态管理：Zustand
- 数据请求：TanStack Query
- 后端：Supabase
- 数据库：PostgreSQL + PostGIS
- 登录：Supabase Auth
- 图片存储：Supabase Storage
- 实时消息：Supabase Realtime

如果目标是高质量 iOS 原生：

- 前端：SwiftUI
- 地图：MapKit
- 后端：Supabase 或 Firebase
- 本地缓存：SwiftData

建议当前阶段优先选 React Native + Supabase，因为 Link Nest 的核心风险在产品流程和社区数据冷启动，跨端快速验证比一开始追求原生细节更重要。

## 4. 信息架构

底部 Tab 建议保留 5 个：

- 发现：附近图书地图和列表。
- 书架：我的共享、借入、出借、收藏。
- 消息：聊天、借阅通知、好友申请、系统通知。
- 邻居：附近用户、好友、附近书友。
- 我的：账户、地址、隐私、历史、设置。

核心用户路径：

1. 游客打开 App，进入发现页。
2. 浏览附近书籍，可切换地图和列表。
3. 进入书籍详情页。
4. 点击申请借阅，触发登录。
5. 登录后选择申请书籍并发送留言。
6. 出借者收到申请，在消息或书架中处理。
7. 双方成为好友或进入聊天。
8. 借阅状态进入进行中，最后完成归还和评价。

## 5. 页面拆解

### 发现模块

- 附近书籍地图页
- 附近书籍列表页
- 书籍详情页
- 出借者详情页
- 搜索与筛选页

关键状态：

- 游客可见：书籍、距离、简介、出借者基础信息。
- 登录后可做：申请借阅、收藏、联系出借者、添加好友。

### 借阅模块

- 申请借阅页
- 申请已发送页
- 借阅状态详情页
- 出借者审批页
- 借阅完成和评价页

借阅状态建议：

- pending：已申请，等待出借者确认
- accepted：已同意，等待交接
- borrowed：借阅中
- return_requested：申请归还
- returned：已归还
- rejected：已拒绝
- canceled：已取消

### 好友模块

- 出借者资料页
- 好友申请页
- 好友申请已发送页
- 好友列表页
- 附近书友页

好友状态建议：

- none：无关系
- pending_sent：我已申请
- pending_received：对方已申请
- friends：已成为好友
- blocked：已拉黑

### 我的书架模块

- 我的书架总览
- 我的共享
- 我的借阅
- 我的收藏
- 添加新书
- 编辑书籍信息
- 书籍发布状态管理

添加图书第一版可以支持：

- 手动输入
- 拍照上传封面
- ISBN 输入

OCR / ISBN 自动查询可以作为第二阶段增强。

### 消息模块

- 消息列表
- 单聊页
- 借阅状态卡片
- 系统通知
- 好友通知

第一版消息可以先做结构化消息：

- 借阅申请通知
- 申请通过/拒绝通知
- 归还提醒
- 好友申请通知

普通聊天可以在 MVP 后半段接入。

### 我的模块

- 个人资料
- 地址和可见范围
- 隐私设置
- 我的评价
- 帮助与反馈
- 关于 Link Nest
- 退出登录

## 7. API / 服务拆分

### Auth Service

- 注册 / 登录 / 退出

- 更新用户资料

### Book Service

- 获取附近书籍
- 获取书籍详情
- 创建书籍
- 编辑书籍
- 上架 / 下架书籍
- 收藏 / 取消收藏

### Borrow Service

- 创建借阅申请

- 拒绝好友申请

### Message Service

- 获取消息列表
- 获取聊天详情
- 发送消息
- 创建系统通知

## 8. 权限与隐私原则

- 游客可以看附近图书，但不能看到精确住址。
- 地图位置需要做模糊化处理，比如偏移到附近街区或只显示大致范围。
- 申请借阅、聊天、添加好友、发布书籍必须登录。
- 默认不公开用户的真实姓名、精确地址和联系方式。
- 双方达成借阅后再展示更具体的交接信息。
- 所有用户生成内容需要支持举报入口。

## 9. 开发里程碑

### 阶段 0：项目初始化，1-2 天

- 创建前端项目。
- 接入路由、主题、基础组件。
- 创建 Supabase 项目。
- 建立数据库 schema 和本地 seed 数据。
- 配置环境变量和基础开发脚本。

交付物：

- App 可启动。
- 底部 Tab 可切换。
- 本地或测试库有示例用户和图书。

### 阶段 1：发现与书籍详情，3-5 天

- 实现发现页地图模式。
- 实现发现页列表模式。
- 实现筛选和排序。
- 实现书籍详情页。
- 实现游客模式和登录拦截。

交付物：

- 未登录用户可以完整浏览附近书籍。
- 点击申请借阅时进入登录流程。

### 阶段 2：登录、书架与发布图书，4-6 天

- 实现登录注册。
- 实现我的书架。
- 实现添加图书。
- 实现编辑图书。
- 实现上架 / 下架。
- 实现图片上传。

交付物：

- 用户可以发布一本可被附近发现的书。

### 阶段 3：借阅申请闭环，4-6 天

- 实现申请借阅。
- 实现申请确认页。
- 实现出借者审批。
- 实现借阅状态流转。
- 实现借入 / 出借列表。

交付物：

- 一次借书流程可以从申请走到归还完成。

### 阶段 4：消息和好友，4-6 天

- 实现消息列表。
- 实现借阅通知卡片。
- 实现基础聊天。
- 实现好友申请。
- 实现好友列表。

交付物：

- 用户可以围绕借阅申请进行沟通，并建立好友关系。

### 阶段 5：我的页面、隐私和 polish，3-5 天

- 实现我的页面。
- 实现地址和范围设置。
- 实现隐私设置。
- 实现错误态、空状态、加载态。
- 完成 UI 统一和动效细节。

交付物：

- App 体验完整，接近内测版本。

### 阶段 6：测试与发布准备，3-5 天

- 单元测试关键服务。
- E2E 测试核心流程。
- 修复权限、状态和边界问题。
- 配置 crash / analytics。
- 准备 TestFlight 或内部测试包。

交付物：

- 可邀请真实用户测试的 Beta 版本。

## 10. 推荐任务顺序

1. 先做静态 UI 和 mock 数据，跑通所有页面。
2. 再接入 Supabase 数据库，替换 mock 数据。
3. 然后做登录和权限拦截。
4. 接着完成书籍发布和附近发现。
5. 再做借阅申请状态机。
6. 最后补消息、好友、设置和测试。

原因是 Link Nest 的核心价值在「附近发现 + 借阅闭环」。先把这条主链路打通，后面的好友、消息、评价都能自然挂上去。

## 11. 第一周建议排期

### Day 1

- 初始化项目。
- 建立设计主题：颜色、字体、间距、按钮、输入框、卡片。
- 建立 Tab 路由和页面骨架。

### Day 2

- 制作发现页地图和列表 UI。
- 准备 mock 书籍和用户数据。

### Day 3

- 制作书籍详情页和出借者详情页。
- 加入申请借阅入口和登录提示。

### Day 4

- 建立 Supabase schema。
- 实现附近书籍查询。

### Day 5

- 接入真实数据请求。
- 修复地图、列表、详情之间的状态和加载体验。

第一周结束时，最好能做到：打开 App 就能看到附近书，点进详情，并清楚知道下一步是申请借阅。

## 12. 主要风险

- 社区冷启动：附近没有书会导致体验空。需要 seed 数据、邀请机制或社区活动。
- 位置隐私：不能直接暴露用户住址，需要做位置模糊化。
- 借阅信任：陌生人借书需要评分、好友、历史记录和举报机制兜底。
- 状态复杂：借阅申请、聊天、好友关系容易互相影响，要先定义清楚状态机。
- 图片和版权：书籍封面来源要谨慎，用户上传更简单安全。

## 13. 下一步

建议下一步直接创建 React Native + Expo 项目，并先实现以下 4 个页面：

- 发现页地图模式
- 发现页列表模式
- 书籍详情页
- 底部 Tab 框架

这 4 个页面完成后，就可以开始把流程图里的主体验变成可点击原型。
