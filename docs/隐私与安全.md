# LinkNest 隐私与安全策略

## 前端原则

- 发现页和详情页只展示模糊距离，例如 `约 1km`，不展示精确地址或经纬度。
- 地图点位只用于表达附近有书，不代表真实住址。
- 具体交接地点应在双方确认借阅后沟通，建议使用公共地点。
- 游客可以浏览可借图书，但收藏、借阅、聊天、发布、好友操作需要登录。
- 书籍详情和出借者资料提供举报入口。

## 数据库原则

- `profiles.home_lat/home_lng` 和 `books.location_lat/location_lng` 不直接展示给客户端 UI。
- `nearby_books` 只返回距离，不要求前端展示精确坐标。
- RLS 已限制：
  - 用户只能更新自己的资料。
  - 用户只能创建/编辑自己的图书。
  - 借阅申请只有借阅双方可读写。
  - 好友关系只有双方可读写。
  - 消息只有会话成员可读写。
  - 收藏只有本人可读写。
  - 举报只有举报人可创建和读取。

## 后续上线前检查

- 打开 Supabase RLS 并逐表测试匿名、普通用户和 service role 权限。
- 举报审核后台必须使用 service role，不能暴露在客户端。
- 生产环境开启邮箱验证。
- 对上传封面做文件大小和类型限制。
- 增加屏蔽用户后隐藏消息、好友申请和借阅申请的逻辑。
